HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Only use h1 once per page.
  • "alt" is needed for 508 compliance.
  • "elements" is the entire element, open to close.
  • "tag" are what appears inside the brackets.
  • HTML stands for Hypertext Markup Language.

CSS

  • CSS stands for Cascading Style Sheets
  • CSS is needed for styling of a document written in a markup language such as HTML.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A wild card "*" will apply to all elements.
  • CSS is a programming language used by developers to define how webpage content is presented to users.
  • External CSS is the preferred option, keeps all CSS rules in a separate file.

Git

  • Git stands for Global Information Tracker.
  • Git is a Version Control System.
  • Used for Branching and Merging.
  • Distributed Development
  • Git is Open Source and Community-driven.
  • Git is a version control system for tracking changes in files, while GitHub is a platform built around Git, providing hosting for repositories, collaboration tools, and project management features.
  • GitHub is cloud-based hosting service.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites.
  • Placing JavaScript near the bottom of an HTML page is one way to accommodate this dependency.
  • Control flow is the order in which a computer executes code in a script.
  • const = variables that cannot change.
  • All line of js must end in a semicolon.
  • To add js file you must add it in the body and at the bottom.
  • Console.log() is used for debugging.
  • Define a function and Call a function.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
